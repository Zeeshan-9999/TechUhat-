2.4 Implementing Context Effectively

Learn how to avoid unnecessary re-renders when using React Context by splitting state and dispatch context, and memoizing values.

Key Takeaways

  • check_circle Always separate state and dispatch into two distinct contexts to prevent consumers that only need to dispatch from re-rendering.
  • check_circle Use useMemo for complex context values, but understand it's not a silver bullet if the dependencies change frequently.

Code Snippet: Split Context

const StateContext = createContext(null);
const DispatchContext = createContext(null);

export function Provider({ children }) {
  const [state, dispatch] = useReducer(reducer, initialState);
  
  return (
    <DispatchContext.Provider value={dispatch}>
      <StateContext.Provider value={state}>
        {children}
      </StateContext.Provider>
    </DispatchContext.Provider>
  );
}
